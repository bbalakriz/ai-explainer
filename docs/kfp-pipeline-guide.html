01 — Building Blocks

KFP v2 DSL Constructs

Twelve core primitives from the official KFP SDK — organized by what they do. Master these and you can build any pipeline.

Components — defining pipeline steps
⚙️
Lightweight

@dsl.component

A hermetic Python function — all imports must be inside the function body. The simplest way to define a step. KFP serializes it into a container at runtime.

@dsl.component(packages_to_install=["numpy"]) def add(a: int, b: int) -> int: import numpy as np # imports inside! return int(np.add(a, b))
📦
Containerized

@dsl.component + target_image

Removes the hermeticity constraint. Can import from adjacent modules. Use kfp component build to bake deps into the image at build time instead of install time.

@dsl.component( base_image="python:3.11", target_image="gcr.io/my-proj/train:v1" ) def train(data_path: str) -> float: from my_utils import run_training return run_training(data_path)
📋
Container

@dsl.container_component

Run any container image with arbitrary commands. Returns a dsl.ContainerSpec — no Python function, just image + command + args.

@dsl.container_component def run_bash(msg: str, out: dsl.OutputPath(str)): return dsl.ContainerSpec( image="alpine", command=["sh", "-c"], args=[f"echo {msg} > {out}"] )
Pipeline — wiring & compiling
🔗

@dsl.pipeline

Wire components into a DAG. Pass task.output or task.outputs["key"] between steps to create dependencies. Pipelines can also nest inside other pipelines.

@dsl.pipeline(name="my-pipeline") def my_pipeline(param: str = "default"): s1 = load_data(path=param) s2 = train(data=s1.output) s2.after(s1) # explicit ordering s2.set_retry(num_retries=3) s2.set_cpu_limit("4") s2.set_memory_limit("16Gi")
📜

compiler.Compiler().compile()

Compiles your @dsl.pipeline function into a portable YAML IR (PipelineSpec protobuf). This YAML is what you submit to a KFP backend or deploy via kubectl.

from kfp import compiler compiler.Compiler().compile( pipeline_func=my_pipeline, package_path="pipeline.yaml", ) # Produces IR YAML → submit to KFP backend
Data Handling — artifacts & parameters
📁

Input / Output Artifacts

Pass typed artifacts (Dataset, Model, Metrics, HTML, Markdown, ClassificationMetrics) between steps. Access via .path, .uri, and .metadata.

from kfp.dsl import Input, Output, Dataset, Model, Metrics @dsl.component def train( data: Input[Dataset], # read-only artifact model: Output[Model], # pre-constructed, write to it ): with open(data.path) as f: ... save_model(model.path) model.metadata["framework"] = "pytorch"
📄

Parameters (small data)

Simple Python types (str, int, float, bool, list, dict) pass as pipeline/component parameters. Used for config values, thresholds, paths.

@dsl.component def configure( lr: float, epochs: int, tags: list, ) -> str: return f"lr={lr}, epochs={epochs}"
Control Flow — branching, loops, cleanup
🔀

dsl.If / dsl.Elif / dsl.Else

KFP v2's conditional branching. Use with dsl.If() as a context manager to gate which steps execute based on upstream outputs or pipeline params.

with dsl.If(eval_task.output >= 0.85): merge_weights(adapter=sft.output) with dsl.Elif(eval_task.output >= 0.70): retrain(data=data.output, lr=1e-5) with dsl.Else(): alert_team(msg="Score too low")
🔁

dsl.OneOf

Gather a single output from mutually exclusive branches (If/Elif/Else) so a downstream task can consume it. Requires an dsl.Else branch to guarantee exactly one result.

with dsl.If(coin.output == "heads"): t1 = handle_heads() with dsl.Else(): t2 = handle_tails() result = dsl.OneOf(t1.output, t2.output) log_result(value=result)
⚡

dsl.ParallelFor + dsl.Collected

Fan out over a list in parallel with optional parallelism cap. Use dsl.Collected to gather all outputs from the loop iterations into a list for a downstream step.

with dsl.ParallelFor( items=[1, 5, 10, 25], parallelism=2 ) as epochs: t = train_model(epochs=epochs) # Gather all models from loop iterations pick_best(models=dsl.Collected(t.outputs["model"]))
🛡️

dsl.ExitHandler + PipelineTaskFinalStatus

Guarantee a cleanup task runs after the pipeline — even if earlier steps fail. Use PipelineTaskFinalStatus in the exit task to inspect what happened.

from kfp.dsl import PipelineTaskFinalStatus @dsl.component def cleanup(status: PipelineTaskFinalStatus): print(f"State: {status.state}") print(f"Error: {status.error_message}") with dsl.ExitHandler(exit_task=cleanup()): train() # cleanup runs even if this fails
02 — Choose Your Tool

KFP DSL vs Elyra

Two approaches to building ML pipelines. Different strengths for different stages of your workflow.

KFP DSL

Code-First Pipelines

Write pipelines as Python code with full programmatic control over every aspect of execution.

  • ✓ Full conditional branching with dsl.If / Elif / Else
  • ✓ Parallel fan-out with dsl.ParallelFor + dsl.Collected
  • ✓ Type-safe artifact passing (Dataset, Model, Metrics)
  • ✓ Three component types: lightweight, containerized, container
  • ✓ Version control — pipelines are .py files, diffs work
  • ✓ CI/CD integration — compile YAML IR and deploy in automation
  • ✓ Reusable component libraries across teams
  • ✓ Task configs: retry, CPU/memory limits, caching, env vars
Elyra

Visual Notebook Pipelines

Drag-and-drop notebooks in JupyterLab to compose pipelines visually — no DSL to learn.

  • ✓ Visual drag-and-drop pipeline editor in JupyterLab
  • ✓ Run existing .ipynb / .py / .r files as pipeline steps
  • ✓ No KFP SDK knowledge required to start
  • ✓ Built into JupyterLab — no context switching
  • ✓ Quick prototyping of linear workflows
  • ✓ Auto-generates KFP YAML under the hood
  • ✗ No conditional branching (If/Elif/Else) or loops
  • ✗ Limited parameterization and no typed artifacts
Why KFP DSL wins for production

Six advantages that matter at scale

🔒

Type Safety

Typed artifacts and compile-time checks catch data mismatches before your GPU job runs.

📝

Git-Native

Pipelines are .py files — diffs, PRs, blame, and rollbacks all work naturally.

♻️

Reusability

Share components as Python packages. Import and compose across projects and teams.

🔀

Conditionals

dsl.If / Elif / Else + dsl.OneOf — real branching, not just linear chains.

⚡

Parallel Execution

ParallelFor with Collected — run sweeps, multi-model evals, or data shards concurrently.

🚀

CI/CD Ready

Compile to YAML IR, validate, and deploy pipelines from your CI/CD automation.

03 — Decision Guide

When to use what?

Answer two questions to find your tool.

Do you need conditional branching, loops, or typed artifacts?

e.g., dsl.If(score >= threshold), dsl.ParallelFor, Input[Model]

Yes
No

Use KFP DSL

Elyra cannot express any of these. You need code-level control.

Is it a linear chain of notebooks?

notebook A → B → C, no branching

Yes
No

Use Elyra

Fast prototyping, visual editing in JupyterLab.

Use KFP DSL

Complex DAGs need code.

from kfp import dsl, compiler from kfp.dsl import Input, Output, Dataset, Model, Metrics # ── Step 1: Load & prepare the dataset ────────────────────── # Lightweight component: hermetic, all imports inside body @dsl.component(base_image="python:3.11") def load_dataset( dataset_path: str, output_data: Output[Dataset], ): import json data = fetch_and_format(dataset_path) with open(output_data.path, "w") as f: json.dump(data, f) # ── Step 2: Supervised Fine-Tuning ────────────────────────── @dsl.component(base_image="pytorch/pytorch:2.1.0-cuda12.1") def sft_train( train_data: Input[Dataset], base_model: str, learning_rate: float, epochs: int, adapter_out: Output[Model], ): from peft import LoraConfig, get_peft_model model = load_base(base_model) lora = LoraConfig(r=16, lora_alpha=32) model = get_peft_model(model, lora) train(model, train_data.path, lr=learning_rate, epochs=epochs) model.save_pretrained(adapter_out.path) # ── Step 3: Evaluate the fine-tuned model ─────────────────── @dsl.component(base_image="pytorch/pytorch:2.1.0-cuda12.1") def evaluate( adapter: Input[Model], eval_data: Input[Dataset], metrics_out: Output[Metrics], ) -> float: score = run_eval(adapter.path, eval_data.path) metrics_out.log_metric("eval_score", score) return score # float output → used by dsl.If # ── Step 4a: Merge adapter into base model ────────────────── @dsl.component(base_image="pytorch/pytorch:2.1.0-cuda12.1") def merge_weights( adapter: Input[Model], base_model: str, merged_out: Output[Model], ): merged = merge_lora_into_base(adapter.path, base_model) merged.save_pretrained(merged_out.path) # ── Step 4b: Retrain with adjusted hyperparams ───────────── @dsl.component(base_image="pytorch/pytorch:2.1.0-cuda12.1") def retrain( train_data: Input[Dataset], base_model: str, adapter_out: Output[Model], ): model = load_base(base_model) train(model, train_data.path, lr=1e-5, epochs=5) model.save_pretrained(adapter_out.path) # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ # Pipeline: wiring it all together with # dsl.If / dsl.Else for conditional branching (KFP v2) # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ @dsl.pipeline(name="sft-eval-merge-pipeline") def sft_pipeline( dataset_path: str = "gs://bucket/train.jsonl", base_model: str = "meta-llama/Llama-3-8B", threshold: float = 0.85, ): # 1. Load data data_task = load_dataset(dataset_path=dataset_path) # 2. Fine-tune sft_task = sft_train( train_data=data_task.outputs["output_data"], base_model=base_model, learning_rate=2e-4, epochs=3, ) # 3. Evaluate eval_task = evaluate( adapter=sft_task.outputs["adapter_out"], eval_data=data_task.outputs["output_data"], ) # 4. Conditional: dsl.If / dsl.Else (KFP v2 syntax) with dsl.If(eval_task.output >= threshold): merge_weights( adapter=sft_task.outputs["adapter_out"], base_model=base_model, ) with dsl.Else(): retrain( train_data=data_task.outputs["output_data"], base_model=base_model, ) # ── Compile to YAML IR ────────────────────────────────────── compiler.Compiler().compile( pipeline_func=sft_pipeline, package_path="sft_pipeline.yaml", )
